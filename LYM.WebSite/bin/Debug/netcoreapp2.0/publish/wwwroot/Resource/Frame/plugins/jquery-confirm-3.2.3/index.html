快速使用

$.alert

adds one button (okay) if no buttons are specified, this lets the user to close the modal. 如果没有指定按钮，会自动添加一个（okay）按钮，这可以让用户关闭模型 

$.alert({
    title: 'Alert!',
    content: 'Simple alert!',
});

$.confirm

if no buttons are specified, two buttons (Okay & cancel) will be added. 如果没有指定按钮，将添加两个按钮（Okay＆cancel）。 

$.confirm({
    title: 'Confirm!',
    content: 'Simple confirm!',
    buttons: {
        confirm: function () {
            $.alert('确定!');
        },
        cancel: function () {
            $.alert('取消!');
        },
        somethingElse: {
            text: 'Something else',
            btnClass: 'btn-blue',
            keys: ['enter', 'shift'],
            action: function(){
                $.alert('Something else?');
            }
        }
    }
});
Showing prompt using confirm（显示提示）

Simply add form to the content and bind the events you want. （只需将表单添加到内容并绑定所需的事件。）
This form can also be loaded via ajax.（此表单也可通过ajax加载。） 

$.confirm({
    title: 'Prompt!',
    content: '' +
    '<form action="" class="formName">' +
    '<div class="form-group">' +
    '<label>Enter something here</label>' +
    '<input type="text" placeholder="Your name" class="name form-control" required />' +
    '</div>' +
    '</form>',
    buttons: {
        formSubmit: {
            text: 'Submit',
            btnClass: 'btn-blue',
            action: function () {
                var name = this.$content.find('.name').val();
                if(!name){
                    $.alert('provide a valid name');
                    return false;
                }
                $.alert('Your name is ' + name);
            }
        },
        cancel: function () {
            //close
        },
    },
    onContentReady: function () {
        // bind to events
        var jc = this;
        this.$content.find('form').on('submit', function (e) {
            // if the user submits the form by pressing enter in the field.
            e.preventDefault();
            jc.$$formSubmit.trigger('click'); // reference the button and click it
        });
    }
});

$.fn.confirm

If no buttons are defined, the default buttons (okay and cancel) will be added, and default action for okay will be to redirect on the given href. use this.$target to get the clicked element. 如果没有定义任何按钮，将添加默认按钮（okay 和 cancel），默认操作中的okay行为,会给定的href上重定向。 使用this.$target来获取点击的元素。

Lazy open

If you want to create a instance of jconfirm and save it for later use. 如果要创建一个jconfirm的实例并保存以供将来使用。 

var a = $.confirm({
    lazyOpen: true,
});
a.open();
a.close();
a.toggle(); // toggle open close.

NOTE : The $.confirm(), $.dialog() & $.alert() methods are alias of jconfirm().
All three methods indirectly call the jconfirm base function altering the provided options. $ .confirm（），$ .dialog（）＆$ .alert（）方法是jconfirm（）的别名。   所有三种方法间接地调用jconfirm基函数来改变提供的选项。

高级: this.$body is the body div for jquery-confirm. You can find and alter any element at run time. this.$body 用于jquery-confirm的主体。 您可以在运行时找到并更改任何元素。

Buttons按钮

Its all what you need to know about buttons in jquery-confirm v3.
Defining multiple buttons is introduced in v3.定义多个按钮在v3中引入

You can access the button element via this.$$heyThere for heyThere button in the below snippet. （您可以通过 this.$$heyThere访问按钮元素。）
this is to change the button properties at run time.（这是在运行时更改按钮属性。）

Definition（定义）

Buttons can be defined in the following fashion, pretty self explanatory. （按钮可以以下列方式定义） 

$.confirm({
    buttons: {
        hello: function(){
            // button action
        },
        hey: function(){
            this.$$hello.trigger('click'); // 点击其他按钮
            this.buttons.hello.setText('Helloooo'); // 做什么

            this.$$hey.prop('disabled', true); // 当前按钮不能点
            // or
            this.buttons.hey.disable();
        },
        heyThere: {
            text: 'Hey there!',
            btnClass: 'btn-blue',
            keys: ['enter', 'a'],
            isHidden: false, //初始化不隐藏
            isDisabled: false, // 初始化可点
            action: function(){
                // button action.
            }
        },
    }
});

按钮文字

$.confirm({
    buttons: {
        something: function(){
            // here the key 'something' will be used as the text.
            $.alert('You clicked on something.');
        },
        somethingElse: {
            text: 'Something else &*', // Some Non-Alphanumeric characters
            action: function(){
                $.alert('You clicked on something else');
            }
        }
    }
});

Button keys(按键)

Listen to keyup events for individual buttons. A button can listen for multiple keys at a time. (监听单个按钮的按键事件。 一次按钮可以一次监听多个键。)

可用选项:
a to Z, tilde, enter, shift, tab, capslock, ctrl, win, alt, esc, space. 
$.confirm({
    buttons: {
        specialKey: {
            text: 'On behalf of shift',
            keys: ['shift', 'alt'],
            action: function(){
                $.alert('Shift or Alt was pressed');
            }
        },
        alphabet: {
            text: 'A, B',
            keys: ['a', 'b'],
            action: function(){
                $.alert('A or B was pressed');
            }
        }
    }
});

In this example, the YES button is hidden using bootstrap's hide class, thus the user can only use keys to proceed. （在此示例中，使用bootstrap隐藏类隐藏“YES”按钮，因此用户只能使用键继续操作。） 

$.confirm({
    title: false,
    content: 'A very very critical action here! <br> ' +
             'the proceed button is hidden.' +
             'The only way to proceed is to press the key <strong>Y</strong>.<br>' +
             'Press <span style="font-size: 20px;">Y</span> to proceed.',
    buttons: {
        yes: {
            isHidden: true, // 通过css隐藏按钮
            keys: ['y'],
            action: function () {
                $.alert('Critical action <strong>was performed</strong>.');
            }
        },
        no: {
            keys: ['N'],
            action: function () {
                $.alert('You clicked No.');
            }
        },
    }
});

Button functions（按钮功能）

Button functions provide a nice and clean way to alter your buttons in run-time. （按钮功能提供了一个漂亮干净的方式来更改运行时的按钮。）
if this is not enough you can use this.$$<buttonName> to get the DOM element （如果这还不够， this.$$<buttonName>来获取DOM元素） 

$.confirm({
    buttons: {
       buttonA: {
            text: 'button a',
            action: function () {
                this.buttons.buttonB.setText('Changed it!');
                return false;
            }
        },
        buttonB: {
            text: 'button b',
            action: function () {
                this.buttons.buttonA.disable();
                return false;
            }
        },
        reset: function () {
            this.buttons.buttonB.show();
            this.buttons.buttonA.enable();
            this.buttons.buttonB.setText('button B!!');
            // this.buttons.buttonA.hide();
            // this.buttons.buttonA.show();
            return false;
        }
    }
});

Close icon

jQuery confirm uses &times; html entity for this close symbol, however you can use Any icon of your choice (fa, glyphicon, zmdi) （jQuery confirm使用&times;这个关闭符号的html实体，但是你可以使用你选择的任何图标（fa，glyphicon，zmdi））

By default closeIcon is set to null. That means, if buttons are not defined the closeIcon will be shown, else will not be shown. （默认情况下， closeIcon设置为null。 这意味着如果没有定义按钮，则closeIcon将被显示，否则将不显示。）
To explicitly show closeIcon set it to a truthy value and vise versa.（要明确显示closeIcon将其设置为真值，反之亦然）

Turn on closeIcon explicitly（明确使用closeIcon） 

$.confirm({
    closeIcon: true
});

Using other libraries for icons（使用其他库的图标） 

$.confirm({
    closeIcon: true,
    closeIconClass: 'fa fa-close'
});

Handle closeIcon's callback（处理closeIcon的回调）

Control what happens when close icon is clicked.（控制点击关闭图标时会发生什么。）
closeIcon can take in function to handle the button click or you can return a button name. （closeIcon可以处理按钮单击功能，也可以返回按钮名称。） 

$.confirm({
    closeIcon: function(){
        return false; // to prevent close the modal.
        // or
        return 'aRandomButton'; // set a button handler, 'aRandomButton' prevents close.
    },
    // or
    closeIcon: 'aRandomButton', // set a button handler
    buttons: {
        aRandomButton: function(){
            $.alert('A random button is called, and i prevent closing the modal');
            return false; // you shall not pass
        },
        close: function(){
        }
    }
});

Custom width

Jquery-confirm uses bootstrap's grid system for its layout by default. You can simply provide column classes to adjust the modal's width. （默认情况下，Jquery-confirm使用bootstrap网格系统进行布局。 您可以简单地提供列类来调整模态的宽度。）

You can also set responsive layouts.（您还可以设置响应式布局。 引导网格文档） Bootstrap grid docs

instead of typing the whole thing, provide keywords like（提供关键字，而不是输入整个事情）
xlarge/xl equivalent to（相当于） col-md-12
large/l equivalent to （相当于）col-md-8 col-md-offset-2
medium/m equivalent to（相当于） col-md-6 col-md-offset-3
small/s equivalent to （相当于）col-md-4 col-md-offset-4
xsmall/xs equivalent to （相当于）col-md-2 col-md-offset-5 

$.confirm({
    columnClass: 'small'
});
$.confirm({
    columnClass: 'col-md-4'
});
$.confirm({
    columnClass: 'col-md-4 col-md-offset-8 col-xs-4 col-xs-offset-8'
});
$.confirm({
    columnClass: 'col-md-4 col-md-offset-8 col-xs-4 col-xs-offset-8',
    containerFluid: true, // this will add 'container-fluid' instead of 'container'
});

Ajax loading（ajax加载）

With jconfirm you have the power to load content directly when needed via ajax, no extra code. （通过jconfirm，您有权通过ajax直接加载内容，无需额外的代码。）
有两种方法可以通过Ajax加载内容:

  1. Pass in String content the URL with "URL:" prepended.（传入字符串内容带有"URL:"的URL前缀。）
    content: "URL:http://example.com/getData?id=1"
  2. Pass in Function that returns a jQuery ajax promise.（传递函数返回一个jQuery ajax承诺。）
    content: function(){ return $.get(...); }
From v3 onwards, buttons are not disabled until ajax call is complete. （从v3起，在ajax调用完成之前，按钮不会被禁用。）
contentLoaded is called when the ajax is complete. before the content is put in DOM. （当ajax完成时调用contentLoaded。 之前的内容放在DOM中。）

Using the "URL:" prefix（使用"URL:"前缀）

Using the url prefix is the quick way, however has some limitations like you cannot modify the ajax call's method, dataType, etc. （使用url前缀是快速的方法，但是有一些限制，如你不能修改ajax调用的方法，dataType等。）
要使用，请使用"URL:"添加您的网址，如 "URL:http://example.com/file.extension"。

注意: the returned data is set as content automatically before contentLoaded callback is called.（在调用contentLoaded回调之前，返回的数据将自动设置为内容。）

view text.txt 
$.confirm({
    title: 'Title',
    content: 'url:text.txt',
    onContentReady: function () {
        var self = this;
        this.setContentPrepend('<div>Prepended text</div>');
        setTimeout(function () {
            self.setContentAppend('<div>Appended text after 2 seconds</div>');
        }, 2000);
    },
    columnClass: 'medium',
});

Using Ajax promise

This option provides full control over the ajax options and what data is to be inserted. The content takes a function that returns a jQuery promise ($.ajax, $.get, $.post, etc.). In this example a json object is requested, and a part of it is set as content. （此选项可以完全控制ajax选项以及要插入的数据。 内容需要返回一个jQuery承诺（$.ajax，$.get，$.post等）的函数。 在此示例中，请求一个json对象，并将其一部分设置为内容。）

NOTE: the returned data is NOT set as content automatically, you must set the content yourself.（返回的数据不会自动设置为内容，您必须自行设置内容。）

查看bower.json 
$.confirm({
    content: function () {
        var self = this;
        return $.ajax({
            url: 'bower.json',
            dataType: 'json',
            method: 'get'
        }).done(function (response) {
            self.setContent('Description: ' + response.description);
            self.setContentAppend('<br>Version: ' + response.version);
            self.setTitle(response.name);
        }).fail(function(){
            self.setContent('Something went wrong.');
        });
    }
});

Ajax complete callback （ajax请求完毕后回调）contentLoaded

When the ajax call is complete the contentLoaded function is called with arguments Data, Status & Xhr object.
contentLoaded is called before the content is put in DOM （当ajax调用完成时，将使用参数Data，Status＆Xhr对象调用contentLoaded函数。在将内容放入DOM之前调用contentLoaded）
If you want to do stuff after the content has put in DOM, use onContentReady （如果你想在内容放入DOM后做一些事情，可以使用 onContentReady） 

$.confirm({
    content: 'url:text.txt',
    contentLoaded: function(data, status, xhr){
        // data is already set in content
        this.setContentAppend('<br>Status: ' + status);
    }
});

Using the url: prefix method 

$.confirm({
    content: function(){
        var self = this;
        self.setContent('Checking callback flow');
        return $.ajax({
            url: 'bower.json',
            dataType: 'json',
            method: 'get'
        }).done(function (response) {
            self.setContentAppend('<br>Done!');
        }).fail(function(){
            self.setContentAppend('<br>Fail!');
        }).always(function(){
            self.setContentAppend('<br>Always!');
        });
    },
    contentLoaded: function(data, status, xhr){
        self.setContentAppend('<br>Content loaded!');
    },
    onContentReady: function(){
        this.setContentAppend('<br>Content ready!');
    }
});

Using the Ajax promise method

Auto close

Do a action if the user does not respond within the specified time.（如果用户在指定的时间内没有响应，请执行操作。）
This comes in handly when the user is about to do something critical. （当用户即将做一些关键的事情时，这是一个很好的例子。）
The autoClose option takes in a string, like 'confirm|4000' where confirm is the action to trigger after 4000 milliseconds.（自动关闭选项带有一个字符串，如“confirm | 4000”，其中，确认是4000毫秒后触发的动作。）
autoClose的实际示例 

$.confirm({
    title: 'Delete user?',
    content: '如果您没有回复，这个对话框将在6秒内自动触发“取消”',
    autoClose: 'cancelAction|8000',
    buttons: {
        deleteUser: {
            text: 'delete user',
            action: function () {
                $.alert('已删除用户!');
            }
        },
        cancelAction: function () {
            $.alert('已取消');
        }
    }
});
$.confirm({
   title: 'Logout?',
    content: '您的时间已过，您将在10秒内自动注销.',
    autoClose: 'logoutUser|10000',
    buttons: {
        logoutUser: {
            text: 'logout myself',
            action: function () {
                $.alert('用户已注销');
            }
        },
        cancel: function () {
            $.alert('canceled');
        }
    }
});

        }

Callbacks

Get more control over the modal, mainly important for binding events for the modal elements. （对于模态元素的绑定事件，更多地控制模态非常重要。）
contentLoaded callback is called when Ajax loading is used （当使用Ajax加载时调用contentLoaded回调） 

$.confirm({
    title: false,
    content: 'url:callback.html',
    onContentReady: function () {
        // when content is fetched & rendered in DOM（当内容在DOM中获取和呈现时）
        alert('onContentReady');
        var self = this;
        this.buttons.ok.disable();
        this.$content.find('.btn').click(function(){
            self.$content.find('input').val('Chuck norris');
            self.buttons.ok.enable();
        });
    },
    contentLoaded: function(data, status, xhr){
        // when content is fetched（获取内容时）
        alert('contentLoaded: ' + status);
    },
    onOpenBefore: function () {
        // before the modal is displayed.(在模态显示之前。)
        alert('onOpenBefore');
    },
    onOpen: function () {
        // after the modal is displayed.（在模态显示之后。）
        alert('onOpen');
    },
    onClose: function () {
        // before the modal is hidden.（在模态隐藏之前。）
        alert('onClose');
    },
    onDestroy: function () {
        // when the modal is removed from DOM（当模态从DOM移除时）
        alert('onDestroy');
    },
    onAction: function (btnName) {
        // when a button is clicked, with the button name（当点击按钮时，使用按钮名称）
        alert('onAction: ' + btnName);
    },
    buttons: {
        ok: function(){

        }
    }
});
